# Lecture 3 

1. 心理模型 (Mental Models)
   - **中文**：心理模型
   - **英文**：Mental Models
   - **解释**：个体对其环境中事物运作方式的理解，帮助人们生成对系统的描述和解释，预测未来事件。

2. 认知摩擦 (Cognitive Friction)
   - **中文**：认知摩擦
   - **英文**：Cognitive Friction
   - **解释**：用户期望的结果与系统输出之间的不匹配，导致用户挫败感，影响用户体验。

3. 用户中心设计方法 (User Centred Design Approach)
   - **中文**：用户中心设计方法
   - **英文**：User Centred Design Approach
   - **解释**：一种设计流程，强调以用户为中心，包括问题场景、设计与规划、用户研究、执行等步骤。

4. 需求建模技术 (Requirement Modelling Techniques)
   - **中文**：需求建模技术
   - **英文**：Requirement Modelling Techniques
   - **解释**：用于理解和表达用户需求的方法，包括任务分析、用户画像、用户故事等。

5. 用户画像 (User Personas)
   - **中文**：用户画像
   - **英文**：User Personas
   - **解释**：虚构用户的详细描述，用于代表一组具有相似目标、模式和/或态度的潜在用户。

6. 用户故事 (User Stories)
   - **中文**：用户故事
   - **英文**：User Stories
   - **解释**：从用户角度出发的简短陈述，描述用户可能想要使用系统或软件的功能及其原因。

7. 使用场景 (Usage Scenario)
   - **中文**：使用场景
   - **英文**：Usage Scenario
   - **解释**：描述用户在没有透露任何系统特定细节的情况下，期望从系统中获得的信息以及采取的步骤。

8. 功能性需求 (Functional Requirements)
   - **中文**：功能性需求
   - **英文**：Functional Requirements
   - **解释**：系统必须实现的功能，通常通过用户画像或用户故事来表达。

9. 非功能性需求 (Non-functional Requirements)
   - **中文**：非功能性需求
   - **英文**：Non-functional Requirements
   - **解释**：不直接涉及系统的具体功能，但对系统的性能、可靠性、可用性等方面提出的要求。

10. 需求文档 (Documenting Requirements)
    - **中文**：需求文档
    - **英文**：Documenting Requirements
    - **解释**：以特定格式记录系统需求的文档，通常包括系统必须在特定条件下执行的行为。
