# Lecture 5

1. 系统思考 (System Thinking)
   - **中文**：系统思考
   - **英文**：System Thinking
   - **解释**：超越软件的工作范围，考虑影响软件的广泛组件和物理环境。

2. 用例描述 (Use Case Description)
   - **中文**：用例描述
   - **英文**：Use Case Description
   - **解释**：描述一组可能的序列，演员将通过系统实现他们的目标。

3. 线框图 (Wireframes)
   - **中文**：线框图
   - **英文**：Wireframes
   - **解释**：基本设计布局，展示内容（图像、文本、按钮等）的排列，重点在于GUI的不同方面应如何定位，而不是它们的外观细节。

4. 原型 (Prototype)
   - **中文**：原型
   - **英文**：Prototype
   - **解释**：未来系统的工作原理模型，有助于在投入时间和金钱于最终产品之前验证想法。

5. 低保真原型 (Low-fidelity Prototype)
   - **中文**：低保真原型
   - **英文**：Low-fidelity Prototype
   - **解释**：通常是纸质基础的原型，用于早期设计验证和用户反馈。

6. 迭代开发 (Iterative Development)
   - **中文**：迭代开发
   - **英文**：Iterative Development
   - **解释**：通过重复迭代来逐步改进原型，直至达到最终产品。

7. 抛掉原型 (Throwaway Prototyping)
   - **中文**：抛掉原型
   - **英文**：Throwaway Prototyping
   - **解释**：在创建最终产品过程中被丢弃的设计或模型。

8. 进化原型 (Evolutionary Prototyping)
   - **中文**：进化原型
   - **英文**：Evolutionary Prototyping
   - **解释**：基于之前的模型或设计构建最终产品。
